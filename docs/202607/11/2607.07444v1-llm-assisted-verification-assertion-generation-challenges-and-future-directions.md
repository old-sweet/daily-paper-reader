## 摘要
基于断言的验证（ABV）在设计验证（DV）过程中扮演着关键角色。然而，ABV需要验证工程师根据规范手动生成断言，这使得它成为芯片设计流程中耗时的一个阶段。随着大型语言模型（LLMs）的最新发展，研究人员已经开始探索将其作为ABV过程的辅助工具，特别是用于从设计规范生成SystemVerilog断言（SVAs）。在本文中，我们概述了近期的工作，重点介绍了生成SVAs的不同方法。特别是，我们研究了基于LLM的SVA生成，并提出一个核心问题：如何使基于LLM的断言生成变得系统化和质量意识？在解决这个关键问题时，我们在每个挑战结束时提供关键要点，总结了重要的方法论见解，并提供了解决这些挑战的指南和方向，以帮助使用LLMs生成高质量的一组断言。

## Abstract
Assertion-based Verification (ABV) plays a critical role in the Design Verification (DV) process. However, ABV requires substantial manual effort in generating assertion from specification by verification engineers, making it a time-consuming stage in the chip design flow. With the recent development of Large Language Models (LLMs), researchers have started exploring their use as an assistance in the ABV process, particularly for generating SystemVerilog Assertions (SVAs) from design specification. In this paper, we provide an overview of recent works, highlighting the different methods used to generate SVAs. In particular, we investigate LLM-based SVA generation and ask a central question: How can LLM-based assertion generation be made systematic and quality-aware? While addressing this key question, we provide Key Takeaways at the end of each challenge, summarizing the important methodological insights, and also provide guidelines and directions in solving those challenges that can help generate a high-quality set of assertions using LLMs.

## 1. 论文的核心问题与整体含义
- **研究动机**：断言式验证（ABV）是芯片设计验证的关键环节，但人工编写 SystemVerilog 断言（SVA）耗时、易错，且需要深厚的领域知识。近年来大语言模型（LLM）的兴起为自动生成断言提供了可能，但现有方法缺乏系统性和质量评估机制。
- **核心问题**：如何使 LLM 辅助的断言生成过程变得系统化且质量可控？即如何通过有效表示设计信息（自然语言规范 NLS 和 RTL）来生成高质量断言集，并对其正确性、覆盖率和空泛性进行可靠评估？

## 2. 论文提出的方法论
- **论文性质**：本文并非提出全新框架，而是对现有 LLM 辅助断言生成方法进行的系统性综述和挑战分析，并据此给出指导原则。
- **核心思想**：断言生成必须以设计规范为主要输入，RTL 仅作为辅助（如信号映射、形式验证）；必须通过结构化表示（如知识图谱、信号级 JSON）来消除规范歧义；必须引入形式验证工具（FPV）驱动的修正循环；评估不能仅依赖覆盖率，还需考虑空泛性（vacuity）和冗余。
- **关键指导原则**（对应提出的“未来方向”）：
  - **从规范而非 RTL 生成断言**：RTL 可能包含 bug，若从 RTL 生成断言会掩盖错误。
  - **精确信号映射**：断言中使用的信号名必须与 RTL 完全一致，避免引入规范独有的信号导致空泛证明。
  - **消除规范歧义**：使用知识图谱（KG）将端口、寄存器、模块等作为节点，关系作为边，实现结构化表示。
  - **超越覆盖率评估**：引入空泛性度量（vacuity metric）、基于锥形影响（COI）的断言重要性分析。
  - **断言精简**：通过等价性检查去除冗余断言，通过强弱关系分析去除弱断言。
  - **FPV 日志驱动的修正循环**：将 FPV 工具的输出（语法错误、反例等）反馈给 LLM 进行迭代优化。
  - **长上下文处理**：利用知识图谱或信号级增量生成，避免超出 LLM 上下文窗口。

## 6. 论文的主要结论与发现
- **关键结论**：
  1. LLM 辅助断言生成有潜力，但目前存在多个严重质量隐患。
  2. 断言必须源自设计规范，而非 RTL 实现。
  3. 精确信号映射是避免空泛证明的必备步骤。
  4. 结构化规范表示（如知识图谱）能有效减少歧义，提升断言覆盖率和正确性。
  5. 覆盖率指标（语句、分支、功能）本身不足，空泛性分析应成为标准质量度量。
  6. 冗余和弱断言应被自动移除，以压缩断言集而不损失验证能力。
  7. 形式验证工具驱动的迭代修正循环对提高断言通过率至关重要。
- **主要发现**（Case Study 数据支撑）：
