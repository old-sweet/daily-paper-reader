## 摘要
梯形逻辑炸弹（LLB）是可编程逻辑控制器（PLC）程序中的恶意控制逻辑，它在触发激活有效载荷之前一直处于休眠状态，而有效载荷可操纵执行器、伪造传感器读数或拒绝操作员控制。我们观察到，真正的恶意逻辑隐藏在功能块体内，现有梯形图验证器会从其中间表示（IR）中丢弃这些功能块，导致炸弹对验证器不可见。我们提出ESBMC-LLB，它使用ESBMC-PLC+作为验证引擎，并添加了一个建模层，该层暴露功能块逻辑，并将炸弹检测重新表述为一个形式化验证问题：扫描看门狗暴露非终止有效载荷，输出接线暴露执行器伪造有效载荷作为安全违规。k-归纳法给出了所有扫描中无炸弹的无界证明，有界模型检查器返回一个反例即触发器——这保证了签名、异常和CFG分类检测器所缺乏的特性。在公开的Iacobelli 2024数据集上，ESBMC-LLB检测到所有30个炸弹并恢复每个触发器；它还检测到规避CFG分类的自适应触发器（计算型、不透明算术型、多扫描型）。我们还报告了首个在PLC-Defuser的SWaT语料库上的语义模型检查器评估：我们的模拟扩展使整个语料库可解析；在v1.0.0版本上，它检测到149/150个炸弹（99%），零误报，并恢复每个触发器；在具有非线性非终止炸弹的后续版本上，由于SMT求解器超时，检测率降至49%。我们得出结论，语义模型检查和CFG分类是互补的——前者提供无界证明、自适应触发器鲁棒性，并处理布尔/整数和线性模拟逻辑；后者导致非线性模拟非终止，我们界定了各自的优势领域。

## Abstract
A Ladder Logic Bomb (LLB) is malicious control logic in a Programmable Logic Controller (PLC) program that lies dormant until a trigger activates a payload to manipulate actuators, forge sensor readings, or deny operator control. We observe that real malicious logic hides inside function-block bodies, which existing ladder-diagram verifiers drop from their intermediate representation (IR), making bombs invisible to provers. We present ESBMC-LLB, which uses ESBMC-PLC+ as its verification engine and adds a modeling layer that exposes function-block logic and recasts bomb detection as a formal verification problem: a scan-watchdog exposes non-termination payloads, and output wiring exposes actuator-forgery payloads as safety violations. k-induction gives an unbounded proof of bomb-absence across all scans, and the bounded model checker returns a counterexample that is the trigger - guarantees that signature, anomaly, and CFG-triage detectors lack. On the public Iacobelli 2024 dataset, ESBMC-LLB detects all 30 bombs and recovers every trigger; it also detects adaptive triggers (computed, opaque-arithmetic, multi-scan) that evade CFG-triage. We also report the first semantic model-checker evaluation on PLC-Defuser's SWaT corpus: our analog extension makes the full corpus parseable; on v1.0.0, it detects 149/150 bombs (99%) with zero false positives, recovering each trigger; on a later version with nonlinear non-termination bombs, detection drops to 49% as the SMT solver times out. We conclude that semantic model checking and CFG-triage are complementary - the former gives unbounded proofs, adaptive-trigger robustness, and handles Boolean/integer and linear analog logic; the latter leads to nonlinear analog non-termination, and we delineate where each wins.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：可编程逻辑控制器（PLC）运行工业控制系统的安全关键逻辑。攻击者可能插入**梯形逻辑炸弹（LLB）**——一种休眠直到特定触发条件激活才执行恶意有效载荷的控制逻辑片段。LLB能操纵执行器、伪造传感器读数或拒绝操作员控制，且能通过调试和短时间模拟检测。
- **背景问题**：现有梯形图（LD）验证器在构建中间表示（IR）时会丢弃功能块（Function Block）的内部逻辑，导致恶意逻辑对验证器不可见。因此，恶意程序与良性程序在IR上无法区分，验证器无法检测LLB。
- **整体含义**：该论文提出一种将LLB检测转化为形式化验证问题的方法，通过暴露功能块逻辑、添加扫描看门狗和输出接线机制，实现对布尔/整数及线性模拟逻辑炸弹的**无界无炸弹证明**和**自动触发合成**，填补了现有签名、异常和控制流图（CFG）分类检测器的不足。

## 2. 论文提出的方法论

- **核心思想**：利用已有的ESBMC-PLC+形式化验证引擎（基于SMT求解器），在其前端添加一个轻量建模层，使隐藏于功能块体内的恶意逻辑暴露给验证器。然后将炸弹有效载荷与可检查的安全属性（如扫描看门狗断言、执行器互斥属性）关联，使炸弹激活成为可到达的安全违规。验证结果：k-induction证明所有扫描周期均无炸弹（SAFE）；有界模型检查（BMC）返回反例，即为**触发条件（trigger）**。
- **关键技术细节**：
  1. **功能块体翻译**：将用户定义的FUNCTION_BLOCK体解析并转换为GOTO代码（赋值、IF、WHILE、比较等），每扫描周期执行一次，输入非确定性。
  2. **扫描看门狗**：为每个梯级循环插入一个受预算约束的迭代计数器，触发门控的非终止有效载荷变为可达安全违规（仿真真实PLC看门狗）。
  3. **输出接线**：功能块输出引脚赋值给使用它的程序变量，使伪造输出传播到程序中，被安全属性检测。
  4. **用于模拟基准的扩展**：添加REAL类型、非布尔线圈/触点、ST翻译器对不支持构造进行过近似（牺牲健全性换取解析覆盖率）。
- **算法流程**：
  1. 输入：PLC程序（PLCopen XML LD或ST）、YAML格式的安全属性。
  2. 预处理：功能块体翻译 + 扫描看门狗 + 输出接线。
  3. 编码：扫描循环建模为`while(true) { in = nondet(); step(); assert(p); }`，其中输入非确定性（开放世界传感器模型），状态持久。
  4. 验证：ESBMC后端执行k-induction（无界证明）或增量BMC（找违规反例）。
  5. 输出：SAFE（无炸弹，无界保证）或 VIOLATION + 反例（即触发条件，如具体的输入值和扫描索引）。

## 6. 主要结论与发现

1. **功能块体翻译是必要条件**：未修改的ESBMC-PLC+丢弃功能块体IR，导致恶意程序与良性程序无法区分；翻译后炸弹逻辑暴露，可实现检测（RQ1）。
2. **在Iacobelli数据集上实现100%召回率、0%误报率**（30/30检出，29/30安全证明，1个UNKNOWN而非误报），并恢复所有触发器（RQ2）。
3. **通用性**：不仅可检测非终止载荷，还可通过安全属性（如执行器互斥）检测无循环的伪造载荷（RQ3）。
4. **自适应对抗鲁棒性**：5种规避CFG分类的变体（计算、平方、分解、时域累积）全部被检测，而CFG分类代理只检出朴素基线（1/5）（RQ4）。
5. **布尔/整数逻辑标度**：在分类语料库（310程序）上实现100%召回、0%误报，中位运行时间70ms（RQ5）。
6. **真实模拟基准**：首次在PLC-Defuser SWaT语料库上运行语义模型检查器，解析率从~5%提升至100%。在v1.0.0（线性触发器）上检出149/150（99%），零误报；在开发快照（含非线性非终止炸弹）上检出49%（73/150），SMT超时为根本原因（RQ6）。
7. **回归完好**：继承的13个ESBMC-PLC+验证结果全部保留。
8. **互补性**：语义模型检查在布尔/整数和线性模拟逻辑上提供无界证明、自适应鲁棒性；CFG分类在非线性模拟非终止上占优。二者互补而非竞争。

## 7. 优点

## 8. 不足与局限

- **非线性算术限制**：对于非线性非终止炸弹（如`i = i * i`），SMT求解器超时，检测率仅49%。这是SMT求解器的固有弱点，作者承认在该领域CFG分类领先。
- **模拟扩展的健全性损失**：为解析模拟程序引入的过近似（如非确定赋值替代不支持构造）导致在Iacobelli良性集上产生1个假阳性（本应SAFE的程序被报告非SAFE）。作者将模拟配置的健全性限于布尔/整数配置。
- **k-induction不完备**：在Iacobelli数据集上，1个良性程序报告UNKNOWN（非SAFE），表明可能漏报或需更大k值。
- **依赖指定的安全属性**：只能检测那些违反已有属性的载荷；未定义的攻击模式（如不违反互斥但默默改变操作参数的伪造）可能逃逸。
- **自适应对抗实验的CFG代理简化**：仅使用透明文字常量标志，未实际运行PLC-Defuser，可能高估自身鲁棒性。
- **单机实验**：所有时序仅在一台Apple Silicon设备上测得，未验证跨平台稳定性或分布式环境。
- **缺少个体运行时间详细分析**：仅在RQ5报告中位/最大时间，未按程序类型或求解器差异化讨论。
- **未与更多方法进行头对头**：只与Iacobelli和PLC-Defuser做了比较；未与TSV、SymPLC等在同一基准上运行比较（Table 1仅定性对比）。
