## 摘要
现代处理器验证因传统基于变异的模糊测试效率低下而难以达到深层架构状态。我们提出HiFuzz，一种新颖的分层强化学习框架，用结构化的两层生成过程替代变异：程序智能体负责全局布局，基本块智能体负责精确指令填充。为解决奖励稀疏问题，HiFuzz集成了自适应覆盖奖励机制和提供内在反馈的语义感知基本块编码器。在三个真实RISC-V核上的广泛评估表明，HiFuzz在覆盖率和缺陷检测方面显著优于最先进的模糊测试工具。

## Abstract
Modern processor verification struggles to reach deep architectural states due to the inefficiencies of traditional mutation-based fuzzing. We propose HiFuzz, a novel hierarchical reinforcement learning framework that replaces mutation with a structured, two-layer generation process: a Program Agent for global layout and a Basic Block Agent for precise instruction filling. To overcome reward sparsity, HiFuzz integrates an adaptive coverage reward mechanism and a semantic-aware basic block encoder providing intrinsic feedback. Extensive evaluations on three real-world RISC-V cores demonstrate that HiFuzz significantly outperforms state-of-the-art fuzzers in coverage and bug detection.

- 传统硬件模糊测试主要依赖**基于变异**的策略，通过修改已有模板生成测试程序。变异常破坏指令语义，导致大量无效程序被提前丢弃，难以触及处理器深层架构状态。

- 现有生成式方法（如Cascade）虽能构造语法正确的指令流，但依赖**随机启发式**，无法从历史执行中学习，搜索效率低；利用强化学习（RL）面临**动作空间复杂、奖励稀疏、奖励偏置**三大挑战。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：采用**两层层次化强化学习架构**，将程序生成分解为宏观布局和微观指令填充两个子问题，分别用不同RL算法处理；结合**语义感知内在奖励**和**自适应模块级覆盖奖励**缓解稀疏性和偏置。
- **关键技术细节**：
  - **可配置测试生成器**：基于构造式生成（constructive generation），分层配置全局结构（内存大小、基本块数量、分布）和每个基本块的指令类别/终止模式；分离宏观与微观决策，保证语法语义有效性。
  - **双智能体架构**：
    - **Program Agent**（高层）：使用**Rainbow DQN**，输入11维生成统计状态，输出1375种离散全局配置之一（决定内存、基本块数、长度分布等）。
    - **Basic Block Agent**（低层）：使用**PPO**，输入12维状态（剩余指令预算、当前特权级等），输出指令类别分布和终止模式；具体指令由底层约束生成器负责，保持动作空间紧凑。
  - **语义感知基本块编码器（BB Encoder）**：
    - 使用结构化分词器将每条指令拆分为语义子令牌（操作数角色、执行单元元数据、CSR符号名），词汇表仅343个令牌。
    - 编码器为**Bi-LSTM**（双向LSTM，注意力池化，输出96维嵌入），两阶段训练：第一阶段自监督掩码语言建模（MLM）学习汇编语法；第二阶段对比学习，以自定义的**BB-Sim算法**（加权序列对齐+依赖关系奖励）计算微架构感知相似度作为监督。
    - 冻结的编码器在模糊测试中提供**即时内在奖励**：嵌入空间中的在线聚类距离 `Rint = min_k (1 - cos(E(bb), C_k))`，无需等待RTL仿真。
  - **自适应覆盖奖励机制（ACRM）**：
    - 将覆盖分解至**模块级**（如`core.div`、`dcache`、`fpuOpt`等），基于**UCB多臂老虎机**动态调整各模块权重。

    - 最终外部奖励 `Rext = Σ w_i·ΔCov_i`，传递给Program Agent。

- **结构优势**：分层设计使智能体专注于不同抽象层，生成器保持语法正确性；语义编码器提供即时内在信号；ACRM缓解了模块级覆盖掩蔽效应。

## 7. 优点
- **创新性**：首次将分层强化学习系统性地应用于硬件模糊测试的构造式生成，有效解决了动作空间复杂和奖励稀疏难题。
- **方法完备**：同时提供了宏观/微观解耦、语义编码内在奖励、自适应覆盖平衡三种互补机制，形成完整的解决方案。
- **有效性验证充分**：覆盖三个不同复杂度的RISC-V核心（包含Chisel和SystemVerilog），证明跨平台通用性；缺陷检测基于独立基准Encarsia，非自建。
- **生成质量优异**：高Prevalence和高依赖链长度意味着测试程序能更深地刺激微架构状态。

## 8. 不足与局限
- **适用范围受限**：
  - 当前仅评估开源RISC-V核，且限于**单hart**程序生成，不支持A扩展（原子操作），无法测试多核或内存顺序相关漏洞。
  - 依赖**模块级覆盖率信号**；若只能获取全局覆盖，ACRM退化为全局奖励，失去平衡能力。
  - 迁移至其他ISA（如ARM/x86）需重写分词器和约束生成器，且需丰富指令元数据来支持BB-Sim度量。
- **实验方差未充分量化**：论文明确提到覆盖率曲线为**单次24小时运行**，未提供多随机种子的均值和标准差，降低了结果的统计可靠性。
- **对比限制**：与GenHuzz仅作结构对比，未在相同仿真后端做覆盖率直接比较，削弱了相对于该工作的优势论证。
- **额外开销**：RL推理/更新和编码器计算带来额外时间，生成时间占总时间的30.5%（Rocket上），高于Cascade的4.4%；虽然总覆盖更优，但引入了一定运行时代价。
- **编码器选择**：使用Bi-LSTM而非Transformer/RWKV，虽在小数据集上更稳定，但可能牺牲长程依赖建模能力（对基本块内短序列影响有限）。
- **技术细节**：部分参数（如ACRM中β(t)的具体调度方案）在正文中未完整披露，影响完全复现。
