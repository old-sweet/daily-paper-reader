## 摘要
形式化合约对于软件测试和验证至关重要，但编写它们仍然费时且容易出错。LLM 为自动形式化提供了一条有希望的途径：从自然语言规范中合成可执行的断言，从而弥合非正式的开发者意图与正式的可执行规范之间的差距。我们提出了 Monty：一个针对断言的自动形式化框架，它解决了断言有效性期望和自然语言歧义性的挑战。我们的技术基于使用一种新颖的一致性得分度量以及对代码进行测试以验证形式化断言所获得的有效性得分来过滤形式化。我们在从 22 个类似集合的 Java 类中衍生的 541 个断言生成任务上评估了我们的方法，并表明我们的技术比直接使用 LLM 翻译断言更可靠地生成地面真相（平均精确度提升高达 20 个百分点）。

## Abstract
Formal contracts are essential for software testing and verification, yet writing them remains labor-intensive and error-prone. LLMs offer a promising path toward autoformalization: synthesizing executable assertions from natural-language specifications and thereby bridging the gap between informal developer intent and formal executable specifications. We present Monty: an autoformalization framework for assertions that tackles the challenges of expectations of validity of assertions and ambiguity in natural-language. Our techniques are based on filtering formalizations using a novel conformance score metric and validity scores obtained from testing the code against formalized assertions. We evaluate our approach on 541 assertion-generation tasks derived from 22 collection-like Java classes, and show that our technique produces the ground truth more reliably (improving upto 20 points in precision on average) than when using LLMs naively to translate assertions.

- **研究动机**：形式化合约对软件测试与验证至关重要，但手工编写成本高。大型语言模型（LLM）提供自动形式化（autoformalization）的可能，但现有工作常假定断言是有效的，忽略了自然语言的歧义性和用户意图的不确定性。实际中，用户可能编写的是无效断言（用于发现bug），或存在多种合理形式化路径。
- **整体含义**：论文旨在构建一个**高精度、高可靠**的自动形式化框架，使开发者能用自然语言表达模块级规范，并自动生成可信的执行断言，从而支持基于AI的代码开发、单元测试和形式验证。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：结合LLM生成多种候选形式化断言，然后使用**双重得分**（一致性得分 + 有效性得分）进行过滤，并利用**主动学习**进行最终歧义消解。
- **关键技术细节**：
  1. **候选生成**：使用LLM（如Qwen2.5-Coder、GPT-oss）根据NL规范+代码骨架生成多个候选JML断言。
  2. **有效性得分**：通过测试生成器（Randoop）对每个候选进行三阶段检查：
     - 语法检查：编译是否通过。
     - 模糊安全性检查：断言求值是否抛出运行时异常。
     - 模糊语义检查：断言在生成的测试输入上是有效还是无效（即test-valid或test-invalid）。
  3. **一致性得分（Clausal Coverage）**：将候选断言反向翻译为NL描述，然后让LLM判断该描述与原始NL规范之间的双向子句覆盖程度，输出分数[-1,1]，阈值设为0.6。重复多次取平均。
  4. **决策矩阵**：分别从有效候选集合和无效候选集合中选出最高一致性得分的断言（需超过阈值）。这同时返回“最可能的有效形式化”和“最可能的无效形式化”，以捕捉自然语言的歧义。
  5. **主动学习**：当存在两个高一致性但有效性相反的候选时，生成一个区分它们的测试输入，并询问用户（模拟为基于ground truth的自动oracle）来选择正确的形式化。
- **公式/算法流程**（文字说明）：
  - 输入：模块M、方法m、NL规范s、观察器方法集合。
  - 输出：最可能忠实的形式化断言和/或最可能无效的形式化断言。
  - 步骤：LLM生成候选→有效性评分→一致性评分→决策矩阵选择→（若需要）主动学习消歧。
