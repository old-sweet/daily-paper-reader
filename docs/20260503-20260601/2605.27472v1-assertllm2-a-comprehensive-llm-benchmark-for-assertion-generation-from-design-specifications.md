## 摘要
基于断言的验证（ABV）是现代硬件设计的基石，但手动将设计意图转化为形式化的SystemVerilog断言（SVA）仍然劳动密集且容易出错。尽管大型语言模型（LLM）在自动化此过程中显示出潜力，但现有基准仍受限于不切实际的任务设定、弱化的规范输入以及过于简化的评估。为解决这些局限性，我们引入了AssertLLM2，这是一个面向硬件验证中真实断言生成的开源基准。AssertLLM2包含83个真实设计，涵盖13个功能类别。对于每个设计，基准提供了一个结构化的设计规范、一个经过验证的依赖完整的黄金级RTL，以及系统性突变的带缺陷RTL变体。这些支持两种实际场景：缺陷预防（从规范生成断言以防止设计错误）和缺陷狩猎（生成断言以暴露预期行为与错误实现之间的差异）。据我们所知，AssertLLM2是第一个明确使用带缺陷RTL作为输入来评估缺陷检测能力的基准。AssertLLM2进一步采用了更严格的评估框架，涵盖语法有效性、形式可证明性、覆盖率以及基于突变的缺陷检测。我们的基准使得对断言生成进行更真实和广泛的评估成为可能，并为实际硬件验证中的最先进LLM建立了严格的基线。

## Abstract
Assertion-based verification (ABV) is a cornerstone of modern hardware design, yet manually translating design intent into formal SystemVerilog Assertions (SVAs) remains labor-intensive and error-prone. While Large Language Models (LLMs) show promise for automating this process, existing benchmarks remain limited by unrealistic task formulations, weak specification inputs, and oversimplified evaluation. To address these limitations, we introduce AssertLLM2, an open-source benchmark for realistic assertion generation in hardware verification. AssertLLM2 contains 83 real-world designs across 13 functional categories. For each design, the benchmark provides a structured design specification, a verified dependency-complete golden RTL, and systematically mutated buggy RTL variants. These support two practical settings: bug-prevention, where assertions are generated from specifications to guard against design errors, and bug-hunting, where assertions are generated to expose discrepancies between intended behavior and faulty implementations. To the best of our knowledge, AssertLLM2 is the first benchmark to explicitly use buggy RTL as input to evaluate bug-detection capability. AssertLLM2 further adopts a more rigorous evaluation framework spanning syntactic validity, formal provability, coverage, and mutation-based bug detection. Our benchmark enables a more realistic and extensive assessment of assertion generation and establishes rigorous baselines for state-of-the-art LLMs in practical hardware verification.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在硬件验证中，手动将设计规范转化为形式化的 SystemVerilog 断言（SVA）耗时且易错。虽然大型语言模型（LLM）有望自动化这一过程，但现有基准存在三大根本缺陷：
  - **任务设定不切实际**：许多基准依赖黄金级 RTL 来生成断言，而在真实验证场景中黄金级 RTL 并不可用。
  - **规范化输入质量弱**：现有基准要么使用简化的需求清单，要么使用质量不一的原始文档，难以公平评估 LLM 能力。
  - **评估指标过于简化**：仅关注语法正确性和形式验证（FPV）通过率，忽略了断言是否真正具有验证价值（如覆盖率和缺陷检测能力）。
- **研究意义**：缺乏真实、全面的基准使得 LLM 在硬件验证中的实际能力被高估或误判，阻碍了该领域的发展。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：构建一个更真实、更全面的基准（AssertLLM2），包含83个真实世界、系统级设计，涵盖13个功能类别。每个设计提供：
  - **结构化规范**：通过 LLM 辅助的四个步骤（去冗余、生成规范化草稿、精炼需求描述、人工验证）从原始 PDF 构建，消除多模态瓶颈和不一致性。

## 6. 主要结论与发现

- **表面指标具有欺骗性**：当前 LLM 在语法成功率上普遍超过80%（最强模型接近95%），FPV 通过率也较高（如 Claude 达到 21/38 平均），COI 覆盖率可超过60%。但这些并未转化为同样高的证明覆盖率或缺陷杀死率（平均杀死率仅约10-12%，联合设置下最高不到30%）。
- **不同模型策略差异明显**：GPT-5.2 偏向生成更少但更可靠的断言；Claude-Sonnet-4.5 更探索性，生成更多但质量参差。Allen 基准才能揭示这些差异。
- **结构化规范优于原始规范**：结构化规范能稳定提升模型表现（如 trigonmetric FPU 上 COI 从37%升至75%），避免了原始文档质量带来的干扰。
- **缺陷狩猎设置的双刃剑效应**：有错 RTL 的输入有助于模型聚焦于实际缺陷（如 PRESENT 和 fft 设计），但在大型设计中（如 openMSP430、xgate）可能误导模型跟随错误行为，导致杀死率下降。
- **现有基准严重高估 LLM 能力**：仅靠语法、FPV、COI 等指标会得出模型“看起来很强”的结论，而实际验证价值有限。

## 7. 优点

- **真实性与全面性**：涵盖83个真实系统级设计（远大于以往的模块级或片段级），任务设定紧贴实际验证流程（分为预防和狩猎两种场景）。
- **首次使用有错 RTL 作为输入**：直接评估 LLM 在缺陷检测中的能力，填补了现有基准的空白。
- **严格的多维度评估框架**：引入证明覆盖率、形式覆盖率、缺陷杀死率等新指标，区分了“表面正确”与“实用价值”。
- **结构化规范构建**：通过规范化的输入排除文档质量干扰，使评估聚焦于模型理解设计意图的能力。
- **开源且可复现**：提供了数据集、评估脚本、LLM 提示词、有错 RTL 生成脚本，支持用户扩展。

## 8. 不足与局限

- **突变错误的代表性**：系统性 AST 突变生成的错误可能无法完全模拟真实硬件 bug（如跨模块交互、时序问题等），因此缺陷杀死率可能低估或高估实际缺陷检测能力。
- **结构化规范的构建成本**：虽然提供了结构化规范，但其构建依赖 LLM 和人工审核，可能难以大规模扩展；且用户若使用原始 PDF 则结果波动大。
- **未涵盖全部验证场景**：仅支持两种设置（预防和狩猎），但未涉及回归验证、覆盖导向生成等工业常见场景。
- **形式化工具依赖**：评估依赖 JasperGold 和 Conformal，这些工具的资源消耗（内存/时间）可能极大，且工具设置（如预算）可能影响结果一致性。
- **模型评估的时效性**：论文发布于 2026 年，评估的 LLM 版本可能已过时，结论具有时效性。
- **未报告统计显著性**：尽管有三次运行，但未提供置信区间或显著性检验，难以判断模型间差异是否显著。
- **硬件限制**：仅在一台 CPU 工作站上运行，未探索分布式推理或更大规模实验。
