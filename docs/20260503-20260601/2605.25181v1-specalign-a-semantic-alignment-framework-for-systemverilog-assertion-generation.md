## 摘要
现有基于大语言模型（LLM）的SystemVerilog断言（SVA）生成方法主要关注语法有效性和形式验证结果，而生成的断言与自然语言规范之间的语义对齐难以量化。因此，在缺乏黄金RTL的情况下，产生幻觉或不对齐的SVA会降低可信度并增加调试工作量。本文提出SpecAlign，一个用于LLM生成SVA的语义评估与精炼框架。SpecAlign引入了两个迭代对齐循环，利用基于蕴涵的分类来评估自然语言属性和SVA是否符合设计规范。我们通过思维链提示生成多个推理路径，并通过自一致性投票机制聚合它们，从而改进对齐决策。分析不对齐的断言以生成可操作的精炼反馈。我们进一步定义了一个定量对齐分数，用于衡量迭代间的语义一致性。实验结果表明，SpecAlign能够有效检测语义不一致性，并在不依赖黄金RTL的情况下改进断言对齐，为传统形式验证评估指标提供了可扩展的补充。

## Abstract
Existing Large Language Model (LLM) approaches to SystemVerilog Assertion (SVA) generation primarily focus on syntactic validity and formal verification outcomes, while semantic alignment between generated assertions and natural language specifications remains difficult to quantify. As a result, hallucinated or misaligned SVAs can reduce confidence and increase debugging efforts in the absence of golden RTL. This paper presents SpecAlign, a framework for semantic evaluation and refinement of LLM-generated SVAs. SpecAlign introduces two iterative alignment loops that assess both natural language properties and SVAs against the design specification using entailment-based classification. We improve alignment decisions by generating multiple reasoning paths using chain-of-thought prompting and aggregating them via a self-consistency voting mechanism. Misaligned assertions are analyzed to generate actionable feedback for refinement. We further define a quantitative alignment score to measure semantic consistency across iterations. Experimental results demonstrate that SpecAlign effectively detects semantic inconsistencies and improves assertion alignment without relying on golden RTL, providing a scalable complement to traditional formal verification evaluation metrics.

## 2. 论文提出的方法论

### 核心思想
- 提出**SpecAlign**框架，通过两个迭代对齐循环（属性对齐循环、SVA对齐循环）来评估和精炼LLM生成的SVA，使其与自然语言设计规范语义一致。
- 使用**基于蕴涵的分类**（ENTAILS / CONTRADICTS / UNKNOWN）来量化对齐程度，结合**思维链（CoT）多路径推理**与**自一致性投票**提升分类可靠性。
- 对不一致的断言，生成结构化、可操作的反馈，驱动自动精炼。

### 关键技术细节
1. **两阶段对齐循环**：
   - **Property Alignment Loop**：检查从规范中提取的自然语言属性（P）是否与设计规范（S）对齐。
   - **SVA Alignment Loop**：在生成SVA后，先通过**自然语言摘要提取**将其转换为自然语言描述（N），再检查是否与对齐属性库（或规范）对齐。
2. **Alignment Checking（对齐检查）**：
   - 使用LLM（GPT-5）对每个自然语言描述X_i进行CoT推理，生成k条推理路径，每条输出子判决v_ij（0=CONTRADICTS，1=ENTAILS，2=UNKNOWN）。
   - 通过**多数投票**聚合得到最终判决V_i。若平局，按保守原则依次优先级：CONTRADICTS > UNKNOWN > ENTAILS。
3. **Inconsistency Analysis（不一致分析）**：
   - 对于CONTRADICTS的案例，利用推理轨迹R_i和规范S，生成具体反馈F_i（包括矛盾元素、正确行为、可操作指令），指导LLM重写属性或SVA。
4. **Evidence Extraction（证据提取）**：
   - 对于ENTAILS案例，提取规范中的直接引文和位置作为证据；对于UNKNOWN案例，识别支持/不支持的元素并生成假设列表。
5. **自然语言摘要提取**：
   - 将SVA拆解为时钟、触发、前件、后件、时序算子等组件，翻译成结构化自然语言描述。
6. **对齐分数**：定义`a_t = E/n`，其中E为ENTAILS数量，n为总断言数，用于衡量每轮迭代的语义一致性。

### 公式/算法流程（文字说明）
- 输入：设计规范S、初始属性集P、初始SVA集A。
- 对于每个属性/断言，重复最多3次循环：
  1. 对齐检查 → 输出判决V_i。
  2. 若为ENTAILS或UNKNOWN，提取证据并加入对齐库；若为CONTRADICTS，进入不一致分析生成反馈F_i。
  3. 使用反馈F_i与规范S，通过LLM重写属性/断言，回到步骤1。
- 输出：对齐的属性库、对齐的SVA库、最终对齐分数。

- **SpecAlign能有效检测并减少语义不一致的断言**：在APB上，经过3轮精炼后，矛盾断言从初始的约148个（基线）降为0个，对齐分数从0.02提升至0.15。在UART上，矛盾断言从142个降为6个，对齐分数从0.06提升至0.28。
- **UNKNOWN类别增加**：精炼后大量断言被归为UNKNOWN（APB 376个，UART 227个），说明许多生成断言包含实现细节，但规范未涵盖。SpecAlign将其区分出来，避免错误地判定为ENTAILS或CONTRADICTS，有助于指导规范完善或人工审查。
