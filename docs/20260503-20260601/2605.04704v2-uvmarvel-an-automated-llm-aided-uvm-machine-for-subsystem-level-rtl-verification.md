## 摘要
验证是集成电路（IC）开发中的主要瓶颈，消耗近70%的工作量。尽管通用验证方法学（UVM）通过结构化验证环境提高了复用性，但构建子系统级UVM测试平台和生成高质量激励仍然需要大量手动编码、反复运行EDA工具以及深入的协议和微架构专业知识。我们提出UVMarvel，一个利用大型语言模型（LLM）为子系统级RTL构建UVM测试平台的自动化验证框架。UVMarvel引入了一种中间表示（IR）和总线协议库，将异构规范转换为协议正确的子系统级UVM测试平台，并使用信号跟踪器和Verilog补丁库指导基于LLM的激励优化。UVMarvel是首个能够自动构建跨主流总线协议的子系统级UVM测试平台的框架，它实现了平均95.65%的代码覆盖率，同时将验证时间从数个人工工作日缩短到4.5小时的自动化执行。

## Abstract
Verification presents a major bottleneck in Integrated Circuit (IC) development, consuming nearly 70% of total effort. While the Universal Verification Methodology (UVM) improves reuse through structured verification environments, constructing subsystem-level UVM testbenches and generating high-quality stimuli still require extensive manual coding, repeated EDA tool runs, and deep protocol and micro-architectural expertise. We present UVMarvel, an automated verification framework that leverages Large Language Models (LLMs) to build UVM testbenches for subsystem-level RTL. UVMarvel introduces an Intermediate Representation (IR) and a Bus Protocol Library to translate heterogeneous specifications into protocol-correct subsystem-level UVM testbenches, and employs a Signal Tracker and a Verilog Patching Library to guide LLM-based stimuli refinement. UVMarvel is the first framework capable of automatically constructing subsystem-level UVM testbenches across mainstream bus protocols, and it achieves an average code coverage of 95.65%, while reducing verification time from several human working days to a 4.5-hour automated execution.

### 方法论

- **核心思想**：利用LLM的语义理解能力，结合结构化中间表示（IR）和领域知识库（总线协议库），实现从异构规范到UVM测试平台的自动构建；再通过信号跟踪器和Verilog补丁库提炼与未覆盖点相关的设计稠密片段，引导LLM生成针对性激励，迭代提升覆盖率。
- **关键技术细节**：
  1. **中间表示（IR）**：将多模态规格文件（文字、时序图、架构图）统一为五部分：模块名、接口描述、寄存器配置、时序特性、功能描述。为LLM提供清晰、验证导向的结构化输入。
  2. **总线协议库（Bus Protocol Library）**：为主流协议（APB、AHB、AXI、P-Channel、Q-Channel）提供UVM骨架（interface、driver、monitor、agent），LLM仅需将IR中的信号名、位宽等参数填入预定插槽，协议控制流固定。实验表明使用该库可将生成成功率从低于7%提升至90%以上。
  3. **覆盖率分析器（Coverage Analyser）**：将VCS生成的HTML覆盖率报告转换为结构化文本，提取未覆盖项及上下文，供LLM使用。
  4. **信号跟踪器（Signal Tracker）**：基于未覆盖点，通过跨文件递归追踪，收集所有与该信号相关的RTL语句和端口连接，构建依赖路径，并映射回顶层可控接口。
  5. **Verilog补丁库（Verilog Patching Template Library）**：将信号跟踪器提取的碎片化语句（如丢失case/always上下文）通过模板（assign、always、case块）补全为语法完整的“过滤DUT”，保留关键依赖链，减少LLM理解负担。
  6. **LLM引导的序列执行**：使用三种LLM（GPT-4.1、Claude 4.5、Gemini 2.5 Pro）生成新序列或豁免点，在原始DUT上模拟，反馈错误信息迭代修复，直到覆盖率收敛。
- **流程**：阶段a（测试平台构建）：IR + 总线协议库 → LLM生成UVM组件 → 仿真 → 初始覆盖率报告。阶段b（激励优化）：覆盖率分析 → 信号跟踪 → Verilog补丁 → LLM生成序列 → 重新仿真 → 迭代。整个流程完全自动化，无需人工干预。
