# Satisfiability Modulo Extensional Constant Arrays 

Mathias Preiner (B) , Aina Niemetz  , and Clark Barrett 

Stanford University, Stanford, USA 

{preiner,niemetz,barrett }@cs.stanford.edu 

Abstract. Reasoning about array data structures is a key requirement for many applications in hardware and software verification, especially in combination with machine integers. The Satisfiability Modulo Theories (SMT) theory of extensional arrays provides array read and write oper-ators and allows extensionality over arrays. This is sufficient to express many aspects of computer-aided verification, but lacks succinctness to efficiently deal with arrays that are initialized with a default value. Exist-ing procedures for extending the SMT-LIB theory of arrays with support for constant arrays are limited to arrays with infinite index domains, and existing implementations in SMT solvers only support a fragment of the theory for finite index domains. In this paper, we present a novel decision procedure for the theory of arrays with constant arrays that supports ar-bitrary index domains and is not limited to the infinite case. We present our procedure as an abstract calculus and show its refutational and sat-isfiability soundness. We implement a decision procedure based on our calculus in the state-of-the-art SMT solver Bitwuzla and evaluate its performance on a diverse collection of benchmarks and use cases. 

# 1 Introduction 

Arrays are an essential data structure in programming languages, and are com-monly used to model memory in hardware. The ability to efficiently reason about arrays, especially in combination with machine integers, is therefore a core re-quirement for many applications of computer-aided verification [6, 15, 28]. Sat-isfiability Modulo Theories (SMT) solvers for the theory of arrays provide such reasoning and serve as the back end engine for a wide range of applications. The SMT-LIB standard [8] defines a theory of arrays with extensionality as originally axiomatized by McCarthy [23]. This theory provides array read (select ) and write ( store ) operators and allows extensionality (equality) over arrays. While this is sufficient to express many aspects of modeling memory and array data structures in hardware and software, the theory lacks the ability to succinctly express operations over multiple indices and memory initialization. When modeling an array initialized with a default value, e.g., to represent a zero-initialized chunk of memory, this initialization must be expressed via a nested store expression that explicitly writes the default value to all relevant indices.

This leads to inaccuracies in the modeling as indices can only be explicitly over-written, and to inefficiencies in solving procedures as the nesting depth of such store chains can become very deep, depending on the number of relevant indices. Extending the SMT-LIB theory of arrays with so-called constant arrays , i.e., arrays that store a single element at every index, provides a succinct way to model array initialization with a default value (which may be an arbitrary term). Existing procedures for this extended theory of extensional constant arrays, however, are all restricted to infinite index domains [20, 26, 34]. In this paper, we present a novel decision procedure for the theory of exten-sional constant arrays that is not restricted to infinite index sorts. Our procedure is presented as an abstract calculus and is a generalization and extension of the 

Lemmas on Demand (LOD) procedure for the extensional theory of arrays pre-sented in [12]. In particular, we make the following contributions :

– We generalize the LOD procedure for the extensional theory of arrays with fixed-size bit-vectors presented in [12] to arbitrary index and element sorts. 

– We present this generalization as an abstract calculus AEXT , which serves as the core calculus of our new decision procedure and is decoupled from the LOD solving paradigm [9, 24]. 

– We then extend this core calculus with support for extensional constant arrays, and prove refutational and satisfiability soundness of our resulting calculus CAEXT for the theory of extensional constant arrays. 

– We implemented our procedure in the state-of-the-art SMT solver Bitwu-zla [27]. We evaluate it on a diverse collection of benchmarks and use cases, including applications in hardware model checking and smart contract ver-ification of the Ethereum virtual machine. Our evaluation shows that our new procedure significantly expands the solving capabilities of Bitwuzla. 

Related Work. The extension of the theory of arrays with constant arrays is not yet standardized in SMT-LIB [8]. Over the years, several array procedures incorporated support for constant arrays to some extent, with limitations [20,26, 31, 34]. Several state-of-the-art SMT solvers [7, 13, 14, 16, 25, 27, 28] have limited support for constant arrays. To the best of our knowledge, none provide full 

support for extensional constant arrays with finite index domains. In 2001, Stump et al. [34] axiomatized the extensional theory of arrays with constant arrays and presented a sound and complete decision procedure for the extensional theory of arrays. The authors also proposed an extension of the procedure with support for constant arrays over infinite index domains, which is not implemented in any state-of-the-art SMT solver. In 2009, de Moura and Bjørner [26] proposed a procedure for the so-called combinatory array logic (CAL) extension of the extensional array theory, which is implemented by the SMT solver Z3 [25]. The CAL extension includes constant arrays, and the procedure is, again, limited to infinite index domains. While Z3 accepts problems that use constant arrays with finite index domains, in our experiments, we encountered cases where it reported wrong answers. All of these instances involved extensionality over constant arrays with finite index domains. We faced a similar issue with MathSAT5 [14], which supports constant arrays and accepts input problems over extensional constant arrays with finite index domains but reported wrong answers for some instances involving extensionality. cvc5 [7] implements a procedure based on [26], restricting support for con-stant arrays to the non-extensional case, and to interpreted constants as elements (arbitrary terms are not allowed). SMTInterpol [13] supports satisfiability and interpolation queries for the extensional theory of arrays with constant arrays as presented by Hoenicke and Schindler [20] in 2019, also limited to the infinite 

index domain case. Yices2 [16] does not have native support for constant arrays, but has limited support for non-extensional constant arrays by means of encoding them as lambda terms. However, Yices2 gives up if the lambda terms cannot be eliminated via beta reduction. Boolector [28] generalizes the lemmas on demand procedure for extensional arrays described in [12] to non-recursive first-order lambda terms [30, 31] and allows reasoning about constant arrays (modeled as lambda terms) for the non-extensional theory of arrays with constant arrays. Bitwuzla [27] implements and extends the array procedure from [12] with support for (extensional) nested arrays and non-extensional constant arrays. Until now, extensionality over constant arrays was not supported.

# 3 A Theory of Extensional Constant Arrays 

The SMT-LIB theory of extensional arrays TA defines a parameterized sort (σ → τ ) for arrays with index sort σ and element sort τ . Its signature contains two interpreted function symbols for accessing ( select ) and writing ( store ) to arrays as axiomatized by McCarthy in [23]. In the following, we use symbol σ for array index sorts, symbol τ for array element sorts, symbols a, b, c, d, e for arrays of sort ( σ → τ ), symbols i, j, k for indices of sort σ, and symbols u, v, w for elements of sort τ .We define the theory of extensional constant arrays T⟨A⟩ = ( Σ⟨A⟩ , I⟨A⟩ ) as an extension of TA with an additional interpreted function symbol for construct-ing constant arrays. The signature Σ⟨A⟩ of theory T⟨A⟩ is shown in Table 1. Array access function a[i] reads an element from array a at index i (select ), and array update function a⟨i ◁ u ⟩ constructs a new array that is identical to ar-ray a except for index i where it stores element u (store ). Further, as originally proposed by Stump et al. in [34], Σ⟨A⟩ includes a constant array constructor 

operator ⟨v⟩σ . This operator creates an array of sort ( σ → τ ) that stores a single element v at all indices and is currently not defined in SMT-LIB. The majority of SMT solvers that support constant arrays, however, support the ((as const (Array IndexSort ElementSort)) v) SMT-LIB construct as a constant array constructor. We omit σ from ⟨v⟩σ if the index sort is clear from the context. 

3.1 Axiomatization of T⟨A⟩ 

The theory of extensional constant arrays T⟨A⟩ is axiomatized by the following four axioms. Array access and update are defined by McCarthy’s read-over-write 

axioms row-eq and row-ne [23]. Note that the axiomatization scheme for the non-extensional theory of arrays only includes these two axioms. Axiom row-eq states that if array a⟨i◁u ⟩, which results from storing element u

at index i in array a, is accessed at that index, the accessed element must be u.

∀ a, i, j, u . i ≈ j =⇒ a⟨i ◁ u ⟩[j] ≈ u (row-eq) Axiom row-ne covers the symmetric opposite case and states that all elements that were not overwritten by store a⟨i ◁ u ⟩ remain unchanged. 

∀ a, i, j, u . i ̸ ≈ j =⇒ a⟨i ◁ u ⟩[j] ≈ a[j] (row-ne) The extensionality axiom ext states that two arrays are equal if and only if both store the same element on each index. The axiomatization scheme for the extensional theory of arrays TA consists of this and the previous two axioms. 

∀ a, b . a ≈ b ⇐⇒ ∀ i . a [i] ≈ b[i] (ext) The axiomatization scheme for the extensional theory of constant arrays T⟨A⟩ 

extends the scheme for TA with the read-over-constant-array axiom roc [34]. This axiom states that accessing a constant array at any arbitrary index always yields its default value (the value it was constructed with). 

∀ i, v . ⟨v⟩[i] ≈ v (roc) 

3.2 Infinite vs. Finite Constant Arrays 

A major limitation of previous work on extensionality of constant arrays [20,26, 34] is its restriction to infinite index domains. In the following, we illustrate why the restriction to infinite constant arrays significantly simplifies procedures for extensional constant arrays. Given two constant arrays ⟨v⟩ and ⟨w⟩, from axioms ext and roc it follows that ⟨v⟩ and ⟨w⟩ are equal if and only if their default values v and w are equal, i.e., ∀v, w . ⟨v⟩ ≈ ⟨ w⟩ ⇐⇒ v ≈ w. For constant arrays with infinite index domain , the same also holds for equality over array stores on constant arrays, e.g., ⟨v⟩⟨ i ◁ w ⟩⟨ j ◁ w ⟩⟨ k ◁ w ⟩ ≈ ⟨ w⟩. Intuitively, infinite constant arrays with different default values cannot be “made” equal since this requires an infinite 

number of array stores, which cannot be expressed in the array theory. Thus, for infinite constant arrays ⟨v⟩ and ⟨w⟩, an equality over update sequences on ⟨v⟩

and ⟨w⟩ cannot hold if v ≈ w does not hold. For finite index domains , however, two constant arrays with different default values can be made equal if sufficiently many stores ensure that they are equal on every index. For example, formula ⟨v⟩⟨ i ◁ w ⟩⟨ j ◁ w ⟩ ≈ ⟨ w⟩ ∧ v̸ ≈ w is satisfiable for constant arrays with an index domain of size two, since it is possible to find an interpretation for i and j such that all indices of ⟨v⟩ are updated to w.More generally, two finite constant arrays ⟨v⟩σ and ⟨w⟩σ with different default values v and w differ on a finite number of indices n = |σI |. # 4 Calculus 

In this section, we first generalize and formalize the decision procedure for the extensional theory of arrays presented in [12], which we refer to as DP A, as an abstract base calculus AEXT . We then extend AEXT with a set of new rules to handle extensional finite and infinite constant arrays, yielding a new calculus 

CAEXT , which maintains full generality with respect to the finiteness of the index domain of constant arrays. Finally, we prove that CAEXT is sound and complete for the theory of extensional constant arrays T⟨A⟩ .In the following, we briefly introduce common assumptions, definitions and notation used to define our base calculus and its extension. 

Assumption 1 Whenever we refer to a set of ΣA-formulas A, we assume that all literals in A are flat . ⊓⊔

This assumption is without loss of generality as any set of ΣA-formulas can be easily transformed into a TA-equisatisfiable set by the addition of fresh variables and equalities. Note that some of the derivation rules we introduce later intro-duce non-flat literals in A. In such cases, we assume that similar transformations maintain the invariant that all literals in A are flat. 

Definition 1 (Configuration). A configuration is either the distinguished con-figuration unsat or a tuple ⟨A, I, π ⟩, where A is a set of ΣA-formulas, I is an interpretation in the empty theory TE that satisfies A, and π is a map of propa-gation steps (as defined in Definition 2). ⊓⊔

Calculus rules modify configurations and are given in guarded assignment form , where the rule premises describe the conditions on the current configu-ration under which the rule can be applied. The conclusion of a rule is either  

> unsat

or otherwise describes the resulting modifications to the configuration. An application of a rule is redundant if it has a conclusion where each component of the derived configuration is equal to the corresponding component in the premise configuration, i.e., it remains unchanged. A configuration is saturated with re-spect to a set of rules if every possible application of each rule is redundant. In this section, we generalize and formalize the LOD decision procedure DP A

for the extensional theory of arrays of [12] as our base calculus AEXT .The main concept of DP A as presented in [12] is based on a fixed point al-gorithm for propagating and congruence checking array reads. The algorithm produces lemmas that are derived from McCarthy’s array axioms without intro-ducing new array reads. In [12], procedure DP A assumes that TA is combined with a single, decidable theory for array indices and elements, and is presented in the context of combining TA with the theory of fixed-size bit-vectors TBV .It is based on the lemmas on demand paradigm [9, 24], but with a bit-vector abstraction at its core, and relies on necessary formula transformations during preprocessing that introduce virtual reads for array equalities and writes. The authors show that for this combination of theories, DP A is sound and complete. We generalize DP A to combinations of TA with arbitrary theories for array indices and elements and decouple it from the LOD paradigm. Our generalized procedure for the theory of extensional arrays TA does not rely on necessary formula transformations during preprocessing, does not assume that array index and element sorts belong to the same theory, and does not rely on the existence of a decision procedure for index and element theories. Instead, it assumes an interpretation of index and element terms in the empty theory TE .We formalize our generalized procedure in terms of our base calculus AEXT .Given a set of ΣA-formulas A and a current interpretation I | = A, the derivation rules of our base calculus aim to propagate information about array indices and elements from one array to another, based on the axioms of TA. For example, for an array access a⟨j ◁ u ⟩[i], if I | = i̸ ≈ j, from axiom row-ne it follows that a⟨j ◁ u⟩[i] ≈ a[i]. We can therefore propagate the information that reading from a⟨j◁u ⟩

at index i yields the same element as reading from a at index i. Our calculus explores propagations under the current interpretation until unsatisfiability is derived, or all possible propagations have been explored. 4.2 Constant Array Extension 

In this section, we extend our base calculus AEXT with a set of new rules to sup-port extensional constant arrays. The new rules for our extended calculus CAEXT 

are given in Figures 4 and 5. Figure 4 shows the additional conflict rule Roc re-quired for handling reads over constant arrays, while Figure 5 lists the new rules to handle extensional constant arrays. The main challenge of determining satisfiability of equalities between store sequences over different finite constant arrays is that we have to reason about default values for indices that have not been updated by array stores. A straight-forward extension of AEXT would be the introduction of virtual reads for all indices of a constant array, i.e., adding ⟨v⟩σ [i] ≈ v for all i ∈ σI . However, this is not feasible in general, neither for infinite nor for finite index domains. For the infinite case, we would never reach a saturated configuration due to infinitely many virtual reads. And while it may be feasible for reasonably small finite index domains, large domains would require a potentially unfeasibly large number of derivations to reach a saturated configuration. Our extended calculus CAEXT avoids the introduction of virtual reads on constant arrays. Instead, it propagates the default value v of a constant array ⟨v⟩

to array a while keeping track of updated indices I(a, ⟨v⟩) between a and ⟨v⟩.

4.3 Correctness 

In this section, we establish the theorems for refutational and satisfiability sound-ness of our base calculus presented in Section 4.1, and its extension presented in Section 4.2. Refutational soundness ensures that the calculus never reports unsat 

unless the initial set of constraints is unsatisfiable. Satisfiability soundness is the complementary property: if the calculus does not report unsat , then the initial set of constraints is satisfiable. This is often referred to as completeness, but we find the term satisfiability soundness more precise in this context. We prove the refutational and satisfiability soundness of our calculus CAEXT in the appendix. Theorem 1 (Refutational Soundness). If a derivation starting with an ini-tial configuration ⟨A, I, π ⟩ ends in unsat , then A is TA-unsatisfiable. 

Theorem 2 (Satisfiability Soundness). If a derivation starting with an ini-tial configuration ⟨A, I0, π 0⟩ ends in a configuration other than unsat , and every applicable rule is redundant, then A is TA-satisfiable. 

# 5 Integration 

In Section 4, we presented our new calculus CAEXT for the theory of extensional constant arrays T⟨A⟩ as a set of derivation rules that modify configurations start-ing from the initial configuration ⟨A, I0, π 0⟩. Our calculus is independent of the underlying solver architecture and only requires the existence of a procedure that provides an interpretation I in the empty theory TE (rule Interp ). We integrated a decision procedure for T⟨A⟩ based on our new calculus CAEXT 

in the state-of-the-art SMT solver Bitwuzla [27]. Bitwuzla specializes on theo-ries relevant for bit-precise reasoning and supports the theories of fixed-size bit-vectors, floating-point arithmetic, and combinations with arrays and unin-terpreted functions. Until now, it implemented a decision procedure for the the-ory of extensional arrays based on [12], extended with support for nested arrays and non-extensional constant arrays. We replaced this procedure with a new decision procedure based on CAEXT , which seamlessly integrates into the LOD architecture of Bitwuzla. In the context of CAEXT , the bit-vector solver at the core of its LOD architecture is responsible for providing interpretation I [29]. As implemented in Bitwuzla, our procedure incorporates several optimizations .

distinct N . When a conflict between two different constant arrays is detected, i.e., rule CongC applies, we produce a resulting lemma that does not actually contain a quantified formula. Instead, the quantified formula is encoded as a special variant of distinct , which we call distinct N . While distinct (t1, . . . , t k) is true iff terms t1, . . . , t k are pairwise different, distinct N (n, t 1, . . . , t k) is true iff at least n terms are pairwise different, i.e., the set of values assigned to t1, . . . , t k has at least n elements. If n = k, then distinct N (k, t 1, . . . , t k) ≡ distinct (t1, . . . , t k), while for n > k , distinct N (n, t 1, . . . , t k) ≡ ⊥ . Given a finite sort τ for ti and 

k > |τ I |, distinct (t1, . . . , t k) ≡ ⊥ , whereas distinct N (n, t 1, . . . , t k) for n ≤ k is not. We encode the quantified formula in CongC as follows. 

∃i : σ. ^

> k∈I(a, ⟨v⟩)∪I(a, ⟨w⟩)

i̸ ≈ k ≡ ¬distinct N (|σI |, I (a, ⟨v⟩) ∪ I(a, ⟨w⟩)) The intuition here is that this quantified formula is true if there exists an index i

that has not been updated between ⟨v⟩ and a and ⟨w⟩ and a, and is thus not included in I∪ := I(a, ⟨v⟩)∪I(a, ⟨w⟩). For infinite index domains, this will always be ⊤. However, for finite index domains it depends on whether the set of updated indices I∪ includes all possible indices of index domain σI , which can be encoded as distinct N (|σI |, I (a, ⟨v⟩) ∪ I(a, ⟨w⟩)). As an additional optimization, we encode a given distinct N (n, t 1, . . . , t k) con-straint lazily and use a custom decision heuristic that tries to assign different values to terms t1, . . . , t k. If our heuristic fails to assign n unique values to the given terms, we lazily expand the encoding of the constraint until we find n

different values or conclude that distinct N (n, t 1, . . . , t k) ≡ ⊥ .

Propagation. In order to reduce memory overhead, we do not immediately record pair ( r, c ) when constructing π(a, t ) while exploring all possible propagations of reads. Since these pairs are only required for constructing R, we instead replay the propagation steps to produce the full map π containing ( r, c ) only for the array reads involved in a conflict. In this replay phase, propagation is performed in breadth-first-search order, which yields the shortest sequence of propagation steps to the conflict under the current interpretation. 

# 6 Evaluation 

We evaluate the performance of our procedure for the theory of extensional constant arrays as implemented in the SMT solver Bitwuzla [27], denoted as configuration Bitwuzla ⟨A⟩ . Bitwuzla supports the (quantified and quantifier-free) theories of fixed-size bit-vectors and floating-point arithmetic, in combination with arrays and uninterpreted functions. We therefore focus on problems over 

finite constant arrays. We compare our configuration Bitwuzla ⟨A⟩ against its baseline version Bitwu-zla (commit 961ccbc5 [3]), and the SMT solvers cvc5 [7] (version 1.3.2), Z3 [25] (commit 9666915d [5]) and MathSAT5 [14] (version 5.6.16). These four support different fragments of theory T⟨A⟩ . In particular, our configuration Bitwuzla ⟨A⟩ is the only SMT solver that fully supports T⟨A⟩ for finite constant arrays. Note that 

Z3 and MathSAT5 allow the use of finite constant arrays without restrictions, but we encountered soundness issues that suggest that they only support the fragment of T⟨A⟩ without extensionality over finite constant arrays. We exclude Boolector [28] and SMTInterpol [13] from the evaluation, since they do not sup-port extensionality over finite constant arrays. We further exclude MathSAT5 

from quantified benchmark sets since it does not support quantifier reasoning. Support for constant arrays is not yet standardized in the SMT-LIB language. Thus, the SMT-LIB benchmark library [32, 33] does not include benchmarks with constant arrays. We therefore evaluated our approach in different usage scenarios: a) on benchmarks without constant arrays but where reasoning about constant arrays is required internally to Bitwuzla; and b) on a compilation of diverse sets of benchmarks with constant arrays, one crafted and the others stemming from different applications. We use the following benchmark sets. 

smtlib (5,112 benchmarks). The non-incremental benchmarks of the quantified logics ABV , ABVFP and ABVFPLRA of the SMT-LIB benchmark library [33]. These benchmarks do not contain constant arrays but require constant array reasoning in subsolver queries of the model-based quantifier instantiation (MBQI) [19] en-gine implemented in Bitwuzla. This is due to the MBQI procedure asserting the model value of each constant x that occurs in the ground formulas via equalities of the form x ≈ xI . The model value of an array constant is represented as a store chain on a constant-initialized array. This inevitably introduces equalities over constant arrays. We use this set to evaluate how much our new procedure improves quantified reasoning with arrays in Bitwuzla. 

mbqi (4,147 benchmarks). A set of quantifier-free non-incremental benchmarks defined over value-initialized and nested extensional constant arrays with fixed-size bit-vectors. These benchmarks were extracted from subsolver queries of the MBQI solver of Bitwuzla ⟨A⟩ on benchmarks from set smtlib that were unsolved by baseline configuration Bitwuzla due to subsolver queries with extensional constant arrays. We included at most one satisfiable and one unsatisfiable of these queries per corresponding smtlib benchmark. 

k-ind (41 benchmarks, 2,838 incremental queries). A set of quantifier-free incre-mental benchmarks over zero-initialized extensional constant arrays with fixed-size bit-vectors. This set was extracted from all queries with constant arrays generated by the k-induction engine of the model checker Pono [21] on bench-marks of the hardware model checking competitions (HWMCC) 2019–2025 [1, 2, 10, 11]. Similarly to Bitwuzla’s MBQI engine, this model-checking engine pro-duces queries that equate array constants with array model values. 

hevm (34 benchmarks). This set contains benchmarks generated by hevm [4,17], a symbolic execution engine for the Ethereum virtual machine. Benchmarks in this set are encoded over bit-vectors of size 256 in combination with uninter-preted functions and zero-initialized extensional constant arrays.

and MathSAT5 , on the other hand, accept the input, but report wrong answers for a significant number of benchmarks. Further, Bitwuzla ⟨A⟩ rejected 3 (out of 12) Math-SAT5 models and 69 (out of 84) Z3 models. These findings suggest that both

crafted ∀. Overall, only 13 benchmarks that could be solved by at least one solver with the quantifier-based encoding could not be solved by Bitwuzla ⟨A⟩ with the native encoding. On closer inspection of the behavior of Bitwuzla ⟨A⟩ on native benchmarks that were hard to solve, we found that the majority of the solving time is spent on finding an index arrangement via distinct N . We leave improving the lazy expansion scheme for distinct N to future work. 

# 7 Conclusion 

We have presented a new calculus CAEXT for the theory of extensional constant arrays T⟨A⟩ without imposing any limitations on the finiteness of the array index domain. We have shown the refutational and satisfiability soundness of CAEXT .We integrated a new decision procedure for T⟨A⟩ based on CAEXT in the state-of-the-art SMT solver Bitwuzla and evaluated its performance on a diverse set of benchmarks. The new decision procedure enables Bitwuzla to solve problems it was not able to handle before, and thus significantly improves over its baseline. While the integration into Bitwuzla and its evaluation has focused on arrays with finite index domains, our procedure is agnostic to the underlying solver architecture and fully supports the infinite domain case. As future work, we plan to integrate a new array solver based on CAEXT in the SMT solver cvc5.
