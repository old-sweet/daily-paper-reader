## 摘要
基于大语言模型（LLM）的断言生成使得寄存器传输级（RTL）设计的形式验证更加易于使用，但仍存在三个实际问题：生成的属性可能因错误原因通过，不同设计的证明代价差异很大，且失败轨迹难以解释。本文提出了一个轻量级、开源框架，在一个循环中解决这些问题。我们的方法结合了变异引导的细化以拒绝弱断言（包括空洞断言以及无法区分故障行为的断言）、一个求解器选择阶段（利用RTL结构在候选可满足性模理论（SMT）后端中选择）、以及因果叙事综合来解释证明失败的原因。在多种RTL设计中，该框架提高了对生成断言的置信度，降低了固定求解器选择的运行时间变异性，并产生了基于反例轨迹的失败解释。结果表明，质量感知的闭合（而非仅断言生成）是实际LLM辅助形式验证中缺失的关键步骤。

## Abstract
Large language model (LLM) based assertion generation is making formal verification more accessible for Register Transfer Level (RTL) designs, but three practical issues remain. Generated properties can pass for the wrong reason, proof cost can vary widely from one design to another, and failing traces are hard to interpret. This paper presents a lightweight, open-source framework that addresses these issues in one loop. Our method combines mutation-guided refinement to reject weak assertions, including vacuous ones and those that fail to distinguish faulty behaviour, a solver-selection stage that chooses among candidate Satisfiability Modulo Theories (SMT) backends using RTL structure, and causal narrative synthesis to explain why a proof failed. Across diverse RTL designs, the framework improves confidence in generated assertions, reduces runtime variability over fixed-solver choices, and produces failure explanations that remain grounded in the counterexample trace. The results suggest that quality-aware closure, rather than assertion generation alone, is the missing step for practical LLM-assisted formal verification.

## 1. 核心问题与整体含义（研究动机和背景）
背景：基于大语言模型（LLM）生成 RTL 断言降低了形式验证（FV）的门槛，但 LLM 生成的断言存在三个突出问题：
- **弱断言问题**：生成的断言可能因错误原因（如空泛通过、无法检测故障行为）而通过，导致虚假信心。
- **求解器性能波动**：不同设计的证明成本差异极大，固定求解器后端可能导致耗时甚至超时。
- **失败轨迹难解释**：原始反例轨迹长且信号多，难以定位根本原因。

研究动机：现有工作（如 AssertLLM、Spec2Assertion、DeepAssert 等）主要专注于断言生成，但缺少对生成断言质量的有效闭环控制。本文提出一个轻量级开源框架，通过**突变引导精炼**、**架构感知求解器选择**和**因果叙事合成**三个组件，形成一个质量感知的闭环，提升断言可信度、稳定运行时间并生成可解释的失败报告。核心观点：**质量感知的闭合是实现 LLM 辅助形式验证实际应用的关键一步**。

## 2. 方法论：核心思想与关键技术细节
### 2.1 突变引导精炼（Mutation-Guided Refinement, MGR）
- **目标**：拒绝弱断言（包括空断言和无法区分故障行为的断言）。
- **流程**：对原始 RTL 注入合成故障（突变），检查当前断言集是否检测到该故障。若断言通过（即突变未被杀死），则幸存突变反馈给 LLM，要求强化断言，形成对抗性循环。
- **有效击杀率**：ρ_eff = (#Killed) / (#Total − #Invalid)，其中 Invalid 表示编辑后的 RTL 不再是有意义的硬件变体（排除分母）。
- **阈值**：若ρ_eff < 85%（T），则启动新一轮断言精炼；通常稳健断言集会收敛到 90–100%。
- **分类**：每个突变结果分为 KILLED、SURVIVED、INVALID、TIMEOUT/INCONCLUSIVE。

### 2.2 架构感知求解器选择（Architecture-Aware Solver Selection）
- **目标**：根据 RTL 结构特征，在翻译为求解器特定形式之前，预先选择最优 SMT 后端（如 Yices、Bitwuzla、Z3、CVC5）。
- **结构指标**：χ = (DA × W + DM + DR) / IC
    - W：数据通路宽度
    - DA：算术运算密度
    - DM：多路选择器密度
    - DR：存储器总位数
    - IC：索引复杂度
- **工作原理**：高一低 MUX 密度表示控制逻辑，高 DA×W 表示宽数据通路，高 DR/IC 表示存储器敏感行为。通过加权和选择求解器，避免直接构建完整证明实例。

### 2.3 因果叙事合成（Causal Narrative Synthesis, CNS）
- **目标**：当证明失败时，生成简洁、忠实于反例轨迹的自然语言解释。
- **方法**：提取失败属性、紧凑轨迹切片、关键信号变化，引导 LLM 生成解释。强调**忠实性**（正确指出失败周期、信号、机制）而非流畅性。
- **评估指标**：四个二进制指标：正确周期、正确信号、正确机制类、无虚构信号。
