## 摘要
PLCopen XML定义了IEC 61131-3梯形图程序的两种编码格式：一种使用<rung>元素的文本编码，另一种将梯形逻辑表示为localId/refLocalId连接的有向图的图形编码。ESBMC-PLC支持文本格式，但将来自CONTROLLINO、Beremiz和OpenPLC Editor的图形导出解析为空GOTO中间表示，导致验证结果空洞地成功。本文提出Graph-ESBMC-PLC，通过基于DFS的图形LD解析器填补了这一空白。该解析器从左电源轨遍历连接图到每个线圈，将梯形路径提取为布尔触点合取式，并应用三层I/O推理方案。按右电源轨connectionPointIn序列对线圈排序，确保SET线圈在RESET线圈之前处理，匹配IEC扫描周期语义。图形到IR的转换保持ESBMC后端不变。对来自CONTROLLINO/OpenPLC Editor的3个图形LD程序的验证显示，与之前的空IR相比，所有程序都生成了带有非确定性输入和梯形逻辑的完整GOTO IR。所有3个程序在k=2时在70ms内验证为SAFE。11个文本LD基准测试完全保留，无回归。报告了两个没有LD内容或具有不支持定时器语义的Beremiz示例作为发现的局限性。工件位于Zenodo（DantasCordeiro2026graphical，doi:10.5281/zenodo.20699856）。

## Abstract
PLCopen XML defines two encoding formats for IEC 61131-3 Ladder Diagram programs: a textual encoding using <rung> elements, and a graphical encoding that represents rung logic as a directed graph of localId/refLocalId connections. ESBMC-PLC supported the textual format but parsed graphical exports from CONTROLLINO, Beremiz, and OpenPLC Editor into an empty GOTO intermediate representation, causing vacuous verification success. This paper presents Graph-ESBMC-PLC, which closes this gap with a DFS-based graphical LD resolver. The resolver traverses the connection graph from leftPowerRail to each coil, extracts rung paths as Boolean contact conjunctions, and applies a three-tier I/O inference scheme. Ordering coils by rightPowerRail connectionPointIn sequence ensures SET coils process before RESET coils, matching IEC scan-cycle semantics. The graphical-to-IR conversion leaves the ESBMC backend unchanged. Validation on 3 graphical LD programs from CONTROLLINO/OpenPLC Editor shows all produce full GOTO IR with nondeterministic inputs and rung logic, versus the empty IR previously. All 3 verify SAFE at k=2 under 70ms. The 11 textual LD benchmarks are fully preserved, with no regression. Two Beremiz examples with no LD content or unsupported timer semantics are reported as discovered limitations. Artifact at Zenodo (DantasCordeiro2026graphical, doi:10.5281/zenodo.20699856).

### 2. 论文提出的方法论

-   **核心思想**：通过增加一个**图形LD解析器**，将图形化的PLCopen XML连接图翻译成ESBMC-PLC后端能够处理的等价的**文本化梯形逻辑**，从而在不修改验证后端核心（GOTO IR编码器、k-induction引擎、Z3 SMT求解器）的前提下，实现对图形格式程序的有效验证。
-   **关键技术细节**：
    1.  **有向图建模**：将<LD>网络中的元素（左/右电源轨、触点、线圈）及其`localId`/`refLocalId`连接建模为一个**有向无环图（DAG）**，节点代表元素，边代表电流流向。
    2.  **DFS梯形提取算法（Algorithm 1）**：
        -   检测到图形格式后，启动DFS（深度优先搜索）从“左电源轨”节点开始遍历。
        -   遍历到每个“线圈”节点，记录从电源轨到线圈的**所有简单路径**。
        -   每条路径上遇到的触点以“与”逻辑（合取）组合，构成一个梯形路径的布尔表达式。到达同一线圈的多条路径则以“或”逻辑（析取）组合。
    3.  **关键排序：右电源轨顺序**：根据右电源轨（`rightPowerRail`）的`connectionPointIn`列表中`refLocalId`的顺序，确定线圈的执行顺序。这对于SET和RESET线圈的正确处理至关重要，确保SET线圈在RESET线圈之前执行，符合IEC 61131-3的锁存语义。
    4.  **三层I/O推断**：正确识别程序的输入和输出变量。
        -   **第一层**：基于`%IX`（输入）/`%QX`（输出）的硬件地址精确识别。
        -   **第二层**：当无地址信息时，采用启发式规则：仅出现在触点一侧的变量为输入，仅出现在线圈一侧的为输出。
        -   **第三层**：出现在两侧或未分类的变量作为内部状态变量处理。这种处理是保守的，即使分类错误也是过度近似，不会导致不安全的验证结果。
-   **公式与定理**：
    -   **公式 (1)， (2)**：定义了单条路径的表达式（`expr(p)`）和线圈的**组合激励条件**（`energy(q)`），即所有路径表达式的析取。
    -   **公式 (3)**：定义了线圈更新规则，包括普通线圈（`none`）、SET线圈（`set`）和RESET线圈（`reset`）的下一状态逻辑。
    -   **定理4.1 (Soundness)**：声称提取的逻辑与IEC 61131-3的梯形图语义等价。
    -   **定理4.2 (Completeness)**：声称DFS能够枚举连接图中从电源轨到每个线圈的所有唯一且完整的路径。
