## 摘要
PLCverif是CERN开发的、自2019年起投入生产使用的最成熟的PLC形式化验证开源平台。然而，它存在两个根本性限制：不支持梯形图（LD）程序（PLC的主要表示法），以及依赖CBMC作为主要后端，从而将验证限制在有界证明范围内。PLCverif的作者们自己认为ESBMC是合适的后端改进方案。先前的工作建立了ESBMC-PLC（一个支持k归纳的文本LD前端）和ESBMC-GraphPLC（支持图形化PLCopen XML）；它们共同覆盖了具有无界证明的LD，但不支持结构化文本（ST），并且带有定时器/计数器功能块的图形化LD仍然无法验证。本文提出了ESBMC-PLC+，这是一个统一框架，弥补了这两个不足：（1）通过MATIEC IEC 61131-3编译器实现ST/SCL前端，将编译后的C代码路由到ESBMC，并带有非确定性输入建模和YAML属性注入；（2）为图形化LD提供功能块状态语义，扩展DFS解析器，将TON/TOF/TP定时器、CTU/CTD计数器以及R_TRIG/F_TRIG边沿触发器建模为GOTO中间表示中的持久扫描周期状态变量。ESBMC-PLC+是第一个通过单一ESBMC后端支持所有三种主要IEC 61131-3输入格式的开源PLC验证框架，支持通过k归纳实现无界安全证明。与PLCverif的功能比较以及在8个基准程序（包括最多含8个整数定时器的程序）上的实验评估表明，ESBMC-PLC+在匹配PLCverif输入覆盖范围的同时提供了更强的保证。与nuXmv的BDD后端相比，ESBMC-PLC+在定时器程序上快400-2000倍，并在nuXmv BDD在120秒超时的情况下完成了证明。

## Abstract
PLCverif is the most mature open-source platform for PLC formal verification, developed at CERN and in production use since 2019. Yet it has two fundamental limitations: no support for Ladder Diagram (LD) programs, the dominant PLC notation, and reliance on CBMC as its primary backend, which restricts verification to bounded proofs. The PLCverif authors themselves identified ESBMC as the appropriate backend improvement. Prior work established ESBMC-PLC (a textual LD frontend with k-induction) and ESBMC-GraphPLC (graphical PLCopen XML support); together, they cover LD with unbounded proofs but not Structured Text (ST), and graphical LD with timer/counter function blocks remains unverifiable. This paper presents ESBMC-PLC+, a unified framework that closes both gaps: (1) an ST/SCL frontend via the MATIEC IEC 61131-3 compiler, routing C-compiled ST to ESBMC with nondeterministic input modeling and YAML property injection; (2) function block state semantics for graphical LD, extending the DFS resolver to model TON/TOF/TP timers, CTU/CTD counters, and R_TRIG/F_TRIG edge triggers as persistent scan-cycle state variables in the GOTO IR. ESBMC-PLC+ is the first open-source PLC verification framework to support all three major IEC 61131-3 input formats via a single ESBMC backend, enabling k-induction-unbounded safety proofs. A feature comparison with PLCverif and experimental evaluation on 8 benchmark programs, including programs with up to 8 integer timers, shows that ESBMC-PLC+ matches PLCverif's input coverage while providing stronger guarantees. Against nuXmv's BDD backend, ESBMC-PLC+ is 400-2,000x faster on timer programs and completes proofs where nuXmv BDD times out at 120s.

## 1. 核心问题与整体含义

- **研究背景**：可编程逻辑控制器（PLC）广泛应用于核电站、水处理等安全关键系统，形式化验证是确保其正确性的重要手段。现有最成熟的开源PLC验证平台是CERN开发的PLCverif，自2019年投入生产使用。
- **核心问题**：PLCverif存在两个根本性局限：
  - 不支持梯形图（LD）程序——这是北美和日本制造业中最常用的PLC表示法，占比超过60%。
  - 主要后端CBMC仅能提供有界验证（BMC），无法对无限扫描周期进行无界安全证明。其无界后端nuXmv BDD在面对包含整数定时器/计数器状态时存在严重的状态爆炸问题。
- **整体含义**：这些限制使得PLCverif难以覆盖工业实际需求，亟需一个统一框架，能够同时支持IEC 61131-3的全部三种主要输入格式（文本LD、图形LD、ST/SCL），并提供更强的无界证明能力。

## 2. 方法论

- **核心思想**：在已有的ESBMC-PLC（文本LD前端+k-induction）和ESBMC-GraphPLC（图形LD DFS解析器）基础上，新增两个关键组件：（1）通过MATIEC编译器将ST/SCL程序编译为C代码，再接入ESBMC的C前端；（2）为图形LD中的定时器、计数器、边沿触发等6种功能块建立扫描周期级的状态语义模型，扩展DFS解析器以正确生成GOTO中间表示。
- **关键技术细节**：
  - **ST/SCL前端**：MATIEC将ST编译为C；ESBMC-PLC+为该C代码添加扫描循环包装：在每次循环开始时插入非确定性输入采样（使用`nondet_bool()`/`nondet_int()`模拟开放世界传感器），并在循环体内注入用户通过YAML指定的安全属性（编译为`__ESBMC_assert()`）。最后路由到ESBMC的k-induction引擎进行无界证明。
  - **图形LD功能块扩展**：在DFS解析器遇到功能块节点时，执行两个动作：①声明持久状态变量（如定时器累计值、边沿前值等）；②在扫描循环中插入状态更新代码，并将功能块输出引脚作为布尔项嵌入到梯级驱动表达式中。具体编码：
    - TON（延时接通）：`if (IN) et++; else et=0; Q=(et>=PT);`
    - TOF（延时断开）：`if (!IN) et++; else et=0; Q=IN || (et<PT);`
    - TP（脉冲）：使用`running`状态标志，`Q=running;`并在计时完成后复位。
    - CTU/CTD计数器：使用前值变量检测上升沿，`Q=(CV>=PV)`。
    - R_TRIG/F_TRIG：使用前值变量检测边沿，产生单扫描脉冲。
  - **支撑理论**：上述编码基于Ebnenasir的形式化LD语义。
- **算法流程**：算法1给出了扩展后的DFS解析器伪代码，与原有的区别在于第10-13行：遇到支持的功能块时先声明变量、发射更新代码、追加输出引脚变量。

- ESBMC-PLC+是**首个开源统一PLC验证框架**，通过单一ESBMC后端支持所有三种主要IEC 61131-3输入格式（文本LD、图形LD、ST/SCL），并提供k-induction无界安全证明。

## 7. 优点

- **统一性**：首次在同一个开源框架内支持文本LD、图形LD和ST/SCL三种IEC 61131-3语言，无需手动转换。
- **证明能力**：k-induction提供无界安全证明，优于PLCverif默认的CBMC有界验证。
- **性能优势**：SMT比特向量算术在整数定时器状态上比BDD方法快2-4个数量级，解决了nuXmv BDD的关键瓶颈。
- **功能块建模**：对6种标准功能块进行了准确的扫描周期级状态编码，填补了图形LD验证的关键空白。
- **易用性**：YAML属性语言无需时态逻辑专业知识，降低了工程人员的使用门槛。
- **开源可复现**：提供Docker镜像和GitHub仓库，一键复现所有实验。

  - ST基准仅包含1个原创程序，图形LD带功能块仅1个程序，样本量小，可能无法充分暴露问题或证明泛化能力（作者也提及PLCopen ST基准因外部仓库访问问题暂未纳入）。

- **语言支持局限**：

- **工业成熟度**：ESBMC-PLC+是2026年的研究原型，而PLCverif自2019年起在CERN投入生产，成熟度和工业验证差距较大。
