## 摘要
基于大语言模型（LLM）的断言生成使得寄存器传输级（RTL）设计的形式验证更加便捷，但仍存在三个实际问题：生成的属性可能因错误原因通过验证，不同设计之间的证明成本差异较大，以及失败轨迹难以解释。本文提出一个轻量级开源框架，在一个闭环中解决这些问题。我们的方法结合变异引导的细粒度优化以拒绝弱断言（包括空断言和无法区分故障行为的断言）、利用RTL结构在候选可满足性模理论（SMT）后端中进行求解器选择的阶段，以及因果叙事综合来解释证明失败的原因。在多种RTL设计上的实验表明，该框架提升了生成断言的置信度，降低了固定求解器选择的运行时变异性，并产生基于反例轨迹的失败解释。结果表明，质量感知的闭环而非仅断言生成是实用LLM辅助形式验证中缺失的关键步骤。

## Abstract
Large language model (LLM) based assertion generation is making formal verification more accessible for Register Transfer Level (RTL) designs, but three practical issues remain. Generated properties can pass for the wrong reason, proof cost can vary widely from one design to another, and failing traces are hard to interpret. This paper presents a lightweight, open-source framework that addresses these issues in one loop. Our method combines mutation-guided refinement to reject weak assertions, including vacuous ones and those that fail to distinguish faulty behaviour, a solver-selection stage that chooses among candidate Satisfiability Modulo Theories (SMT) backends using RTL structure, and causal narrative synthesis to explain why a proof failed. Across diverse RTL designs, the framework improves confidence in generated assertions, reduces runtime variability over fixed-solver choices, and produces failure explanations that remain grounded in the counterexample trace. The results suggest that quality-aware closure, rather than assertion generation alone, is the missing step for practical LLM-assisted formal verification.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：基于大语言模型（LLM）生成 RTL 形式验证断言虽降低了前端编写断言的负担，但存在三个实际瓶颈：
  - **弱断言**：生成的属性可能由于空虚（vacuously）或无法区分设计错误行为而通过验证，导致误信。
  - **证明成本波动大**：不同设计的最优 SMT 求解器后端差异显著，固定选择可能导致超时或效率低下。
  - **失败轨迹难解释**：反例轨迹冗长，新用户难以快速定位真正失败原因。
- **整体含义**：LLM 辅助形式验证不仅需要断言生成，更需要一个**质量感知的闭环**（quality-aware closure），即在生成后通过自动检查、求解器选择和解释反馈来提升断言的可靠性和实用性。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个包含三个模块的轻量级开源闭环框架：
  - **Mutation-Guided Refinement (MGR, 变异引导优化)**：通过向原始 RTL 注入故障（突变），检查当前断言是否能检测到这些故障。若突变存活（断言未触发），则认为断言强度不足，反馈给 LLM 强化断言，循环直至有效杀灭率达到阈值。
    - 有效杀灭率：ρₑff = #Killed / (#Total − #Invalid)
    - 阈值 T = 85%（实验收敛至 90–100%）
    - 突变结果分类：KILLED、SURVIVED、INVALID（设计无意义变体）、TIMEOUT/INCONCLUSIVE。
  - **Architecture-Aware Solver Selection（架构感知求解器选择）**：在形式验证执行前，根据 RTL 结构计算轻量级特征分数，自动选择最佳 SMT 后端，避免对默认求解器的依赖。
    - 分数公式：χ = (DA × W + DM + DR) / IC
      - W：数据通路宽度；DA：算术密度；DM：多路选择器密度；DR：存储器总比特数；IC：索引复杂度。
    - 使用规则（rule-based policy）基于分数选择求解器，本文实验中对所有基准均匹配 Oracle 最优选择。
  - **Causal Narrative Synthesis (CNS, 因果叙事综合)**：当形式证明失败时，提取关键反例信息（失败属性、关键信号跳变、最相关周期），提示 LLM 生成简洁、基于事实的解释，重点关注忠实度（faithfulness），而非流畅性。
    - 评估标准四个二进制指标：正确失效周期、正确关键信号、正确失效机制、无虚假信号声明。只有四者同时满足才算 faithful。
