## 摘要
现有的大语言模型（LLM）方法在生成SystemVerilog断言（SVA）时主要关注语法有效性和形式验证结果，而生成的断言与自然语言规范之间的语义对齐仍然难以量化。因此，在缺乏黄金RTL的情况下，出现幻觉或不对齐的SVA会降低信心并增加调试工作量。本文提出了SpecAlign，一个用于评估和优化LLM生成的SVA的语义框架。SpecAlign引入了两个迭代对齐循环，通过基于蕴涵的分类来评估自然语言属性和SVA是否符合设计规范。我们通过思维链提示生成多个推理路径，并通过自一致性投票机制进行聚合，从而改进对齐决策。对不对齐的断言进行分析，以生成可操作的反馈用于优化。我们还定义了一个定量的对齐分数，用于衡量迭代间的语义一致性。实验结果表明，SpecAlign能有效检测语义不一致性，并在不依赖黄金RTL的情况下提高断言的语义对齐，为传统形式验证评估指标提供了可扩展的补充。

## Abstract
Existing Large Language Model (LLM) approaches to SystemVerilog Assertion (SVA) generation primarily focus on syntactic validity and formal verification outcomes, while semantic alignment between generated assertions and natural language specifications remains difficult to quantify. As a result, hallucinated or misaligned SVAs can reduce confidence and increase debugging efforts in the absence of golden RTL. This paper presents SpecAlign, a framework for semantic evaluation and refinement of LLM-generated SVAs. SpecAlign introduces two iterative alignment loops that assess both natural language properties and SVAs against the design specification using entailment-based classification. We improve alignment decisions by generating multiple reasoning paths using chain-of-thought prompting and aggregating them via a self-consistency voting mechanism. Misaligned assertions are analyzed to generate actionable feedback for refinement. We further define a quantitative alignment score to measure semantic consistency across iterations. Experimental results demonstrate that SpecAlign effectively detects semantic inconsistencies and improves assertion alignment without relying on golden RTL, providing a scalable complement to traditional formal verification evaluation metrics.

## 一、核心问题与整体含义（研究动机与背景）

- **问题**：现有利用大语言模型（LLM）生成 SystemVerilog 断言（SVA）的方法主要关注**语法有效性**和**形式验证结果**，但生成的断言与自然语言设计规范之间的**语义对齐**难以量化。这导致LLM可能产生“幻觉”或不对齐的断言，在缺乏黄金RTL（golden RTL）的情况下，降低验证信心并增加调试工作量。
- **背景**：形式验证工具（如JasperGold）可以判断断言在RTL上是否被证明或产生反例，但**无法直接回答断言是否反映了自然语言规范中的行为语义**。例如，一个断言虽然语法正确且被形式验证证明，却可能因采样时钟错误而在语义上无效。此外，形式验证面临状态空间爆炸等可扩展性问题，难以大规模评估LLM生成的断言集合。
- **本文目标**：提出一个无需黄金RTL、基于规范的语义对齐评估与细化框架 **SpecAlign**，通过两个迭代对齐循环，定量衡量并改善LLM生成的SVA与自然语言规范之间的语义一致性，作为传统形式验证的可扩展补充。

## 二、方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将SVA生成分为两个阶段：**属性提取**（从规范提取自然语言属性P）和**SVA生成**（将P映射为SVA）。SpecAlign在两个阶段后分别嵌入**属性对齐循环**和**SVA对齐循环**，对生成的属性/断言进行语义对齐检查与迭代细化。
- 利用LLM的**思维链（Chain-of-Thought）推理**生成多个推理路径，通过**自一致性投票**（Self-Consistency Voting）得出对齐分类（Entails/Contradicts/Unknown），提高判断可靠性。
- **无量纲对齐分数** $a_t = \frac{E}{n}$ 衡量每轮迭代中语义蕴含（Entails）断言的比例，指导迭代收敛。

### 关键技术细节

1. **自然语言摘要提取（NL Summary Extraction）**  
   - 将每条SVA $A_i$ 转换为自然语言摘要 $N_i$，使其与自然语言属性 $P_i$ 处于同一语义层次。提取组件包括：触发时钟、disable iff条件、前提（antecedent）、蕴含操作符（|-> 或 |=>）、时序约束、结论（consequent），并输出一句完整描述。

2. **对齐检查（Alignment Checking）**  
   - 对每个自然语言描述 $X_i$（$P_i$ 或 $N_i$），结合设计规范 $S$（或已对齐属性库），用LLM进行**蕴含分类**：
     - **Contradicts（0）**：至少一个元素被规范明确矛盾。
     - **Entails（1）**：所有元素被支持或合理假设不矛盾。
     - **Unknown（2）**：至少一个元素既未被支持也未被矛盾（实现细节未在规范中定义）。
   - 生成 $k$ 个推理路径，每个路径输出子判决 $v_{i,j} \in \{0,1,2\}$，通过**多数投票**（平局时优先级 Contradicts > Unknown > Entails）得到最终判决 $V_i$。

3. **不一致分析（Inconsistency Analysis）**  
   - 对分类为 **Contradicts** 的断言，利用原推理路径 $R_i$ 和规范 $S$，生成**结构化、可行动的反馈** $F_i$，指出矛盾元素、规范中的正确行为，并给出具体修改指令（如调整触发条件、时序等）。

4. **证据提取（Evidence Extraction）**  
   - 对 **Entails** 的断言，提取规范中支持该断言的直接引用和位置，供人工审查。
   - 对 **Unknown** 的断言，识别哪些元素被支持、哪些元素不明确，列出假设条件并给出多种可能解释。

5. **细化（Refinement）**  
   - 使用反馈 $F_i$ 和规范 $S$ 重新生成属性或断言 $X'_i$，要求：仅基于规范和反馈，不引入新信号或行为。然后重新进入对齐检查循环，最多迭代固定次数。

6. **对齐分数**  
   - $a_t = \frac{\text{Entails数}}{n}$，反映第 $t$ 轮迭代中语义蕴含断言的占比，目标是通过细化提升 $a_t$。

### 算法流程（文字描述）
1. 从设计规范 $S$ 提取自然语言属性集 $P$。
2. 使用LLM将 $P$ 生成初始SVA集 $A$。
3. **属性对齐循环**：
   - 对每个属性 $P_i$，执行对齐检查，得到判决 $V_i$ 和推理路径 $R_i$。
   - 若 $V_i$ 为 Contradicts → 执行不一致分析生成反馈 $F_i$ → 细化得到 $P'_i$ → 返回对齐检查。
   - 若 $V_i$ 为 Entails → 添加至已对齐属性库，并提取证据。
   - 若 $V_i$ 为 Unknown → 提取证据，标记待人工审查。
   - 循环至所有属性对齐或达到最大迭代次数。
4. **SVA对齐循环**：
   - 对每个SVA $A_i$，先进行自然语言摘要提取得到 $N_i$。
   - 使用已对齐属性库（或原始规范）作为参考，对 $N_i$ 执行对齐检查、不一致分析、细化（类似属性循环）。
5. 最终输出已对齐的断言集与对齐分数。

## 六、主要结论与发现

1. **语义对齐评估的必要性**：形式验证结论（proven/CEX）不能替代语义一致性判断。大量被证明的断言与规范矛盾，证明正确但语义无效。
2. **SpecAlign能有效降低矛盾断言**：通过迭代细化，APB和UART的矛盾断言分别从148和142降至0和6。
3. **对齐分数提升**：经过3轮迭代，APB对齐分数从0.02提升至0.15，UART从0.06提升至0.28。
4. **Unknown类别重要性**：将未定义行为从矛盾中分离，有助于区分真正的不一致与规范覆盖不足，指导规范完善或人工审查。
5. **不依赖黄金RTL**：提供了一种无需RTL的语义验证途径，可作为形式验证的互补。

## 七、优点（方法/实验设计亮点）

- **新颖视角**：首次明确提出“语义对齐”量化问题，并构建无RTL的评估框架，填补了LLM生成SVA评估中的空白。

- **定量指标**：定义对齐分数，便于监控迭代效果并进行比较。
